三、 目前具備的核心功能與架構 (Current Features)

1. 前端架構與 UI 呈現
*  模組化設計：將狀態標籤抽離為共用元件 OrderStatusBadge.jsx，提升代碼複用率與維護性；實作兩欄式商品詳情頁 (ProductDetail.jsx) 提升視覺層次。
*  防護級 UI 狀態回饋：實作具備企業級防護的 Toast 提示系統，運用 useRef 與 useEffect 的 Cleanup function 徹底消滅連續操作導致的計時器記憶體洩漏與 React 卸載報錯。
*  現代化 UX 動效：全面導入 Tailwind group-hover 與 scale-105 實作高質感商品圖片微互動；加入防呆的空狀態 (Empty State) 視覺引導。

2. 會員與權限防護系統 (Authorization & Security)
*  企業級授權閘門 (Gate)：捨棄容易造成 Artisan 編譯錯誤的閉包中介層，改於 AppServiceProvider 中定義 Gate::define('admin')，並在 web.php 路由統一使用 can:admin 中介層實作雙重防護。
*  前端入口隱藏：Navbar 採用條件渲染，非 admin@pinkpick.com 之使用者絕對無法看見或進入後台介面。

3. 管理員後台系統 (Admin Panel - SPA CRUD)
*  訂單狀態管理：透過 Inertia router.patch 實現無跳轉狀態更新 (待處理/已出貨/已完成)，並搭配滾動保留 (preserveScroll) 優化批次操作體驗。
*  商品維護系統：採用 React Modal 實作一頁式 CRUD。結合 withQueryString() 完美保留搜尋關鍵字與分頁狀態；後端使用 with('category') 徹底解決 N+1 Query 效能瓶頸。
*  圖片上傳處理 (File Upload)：前後端分離的標準解法。前端使用 e.target.files 與 URL.createObjectURL 實作圖片即時預覽；發送更新時採用 router.post 搭配 _method: 'put' 規避 PHP 限制；後端透過 Storage::disk('public') 自動處理新圖儲存與舊圖刪除。

4. 購物車與商品邏輯
*  規格選擇 (EAV 模式) 與嵌套預載入：在詳情頁使用 nested eager loading (attributes.attribute) 一次性拉取規格群組；結帳資料以 selected_attributes 欄位儲存並透過 hash 進行防重複比對。
*  連續購物 UX：商品詳情頁加入購物車後保留已選規格，免去使用者重複點擊的困擾；並將 Navbar 購物車數字實現即時跳動同步。
*  計價權威後端化：所有金額計算均在後端重新撈取 DB 原始定價，徹底阻斷前端修改 Payload 竄改價格的攻擊路徑。
*  Service Pattern 架構封裝：將訪客購物車無縫合併的邏輯抽離至獨立的 CartService。結合悲觀鎖 (lockForUpdate) 與 DB::transaction，完美整合於登入與註冊流程中。

5. 結帳與訂單處理
*  原子性交易防護 (Atomic Transactions)：運用 DB::transaction 確保訂單建立、明細快照寫入與購物車清理三者處於同一個原子操作中，消滅資料不一致風險。
*  單價快照 (Price Snapshot)：結帳時將商品當下售價與規格固定存入 order_items，確保歷史紀錄不受未來商品價格波動影響。
*  閉環資料流 (Data Loop) 與無感儲存：擴充 users 資料表加入 phone 與 address。結帳流程自動將新會員聯絡資訊存為預設值，並串接至「個人資料設定」頁面完美閉環。
