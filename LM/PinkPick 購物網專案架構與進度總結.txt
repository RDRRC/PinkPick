一、 專案基本資訊
- 專案名稱：PinkPick Shopping Network (Demo)
- 開發目標：展示全端開發能力，針對求職面試設計。
- 更新日期：2026-04-02
- 技術棧：Laravel 11.x + React + Inertia.js + Vite
- 開發環境：Laragon (PHP 8.x, MySQL, Apache)
- 資料庫現況：MySQL 已建立，包含商品、分類、購物車、訂單等核心資料表，具備完整關聯約束。

二、 本地開發環境配置 (Laragon)
1. 核心工作：自動生成 http://pinkpick.test 網域、管理 MySQL。
2. 開發 SOP：
    - 啟動 Laragon (Start All)。
    - 終端機執行 npm run dev (啟動 Vite 熱更新)。
    - 瀏覽器訪問商城首頁。

三、 目前具備的核心功能與架構 (Current Features)
- 前端模組化設計：Shop.jsx 作為容器，將 Navbar, FilterSidebar, Pagination, ProductCard 抽離，提升代碼複用率。
- 全域狀態管理 (State Management)：使用 React Context API 建立 CartContext，達成購物車數據在全站（Navbar 到商品頁）的 0 延遲同步。
- 原子性交易防護 (Atomic Transactions)：運用 DB::transaction 確保訂單建立、明細寫入與購物車清理三者處於同一個原子操作中，消滅資料不一致風險。
- 企業級安全防護 (Security)：
    - IDOR (越權存取) 防護：嚴格驗證 Session 與 UserID 歸屬，阻絕非法竄改他人購物車。
    - 計價權威後端化：所有金額計算均在後端重新撈取 DB 原始定價，徹底阻斷前端修改 Payload 竄改價格的攻擊路徑。
- 進階資料庫設計：採用 EAV 模式實作動態規格，並在結帳時實作 Price Snapshot (單價快照)，確保訂單紀錄不受未來價格波動影響。
- SPA 無縫結帳體驗：結合 Inertia.js 的單頁導航與 HTML5 Cross-DOM Form Submission 屬性，優化複雜佈局下的表單提交邏輯。
- 進階防禦機制：實作全域 Error Handler，攔截 500 錯誤並保護底層 SQL 資訊。
