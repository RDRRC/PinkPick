三、 目前具備的核心功能與架構 (Current Features)

  1. 前端架構與 UI 呈現
  - 模組化設計：以 `Shop.jsx` 作為容器，抽離 `Navbar`, `FilterSidebar`, `Pagination`, `ProductCard`，提升代碼複用率。
  - 全域狀態管理：透過 React Context API (`CartContext.jsx`) 統一管理購物車邏輯，並放置於獨立的 `Contexts` 目錄下落實關注點分離。實現 Navbar 與購物車頁面之間 0 延遲同步。
  - SPA 體驗：運用 Inertia.js 達成單頁導航無縫切換，並解決了前端反單引號解析問題，確保路由跳轉順暢執行。
  - 錯誤與防護：實作前端錯誤攔截 (`errorMessage`)，自動拆解 Validation 陣列，並建立保護層避免直接拋出 500 底層 SQL 錯誤。
  - 現代化 UX 設計：導入 Navbar 下拉選單設計以集中帳號操作；針對前端空白數據引入「Empty State Design (空狀態設計)」，提供友善的「去逛逛」按鈕引導。

  2. 購物車與商品邏輯
  - 混合型購物車 (Hybrid Cart)：支援訪客透過 Session ID 將商品加入購物車。當訪客登入時，系統會自動比對並合併 Session 購物車與會員購物車的資料。
  - 規格選擇 (EAV 模式)：實作動態商品屬性與規格 (如顏色、尺寸)，使用者可針對特定規格加入購物車，資料庫以 `selected_attributes` 欄位儲存並透過 hash 進行防重複比對。
  - 計價權威後端化：所有金額計算均在後端重新撈取 DB 原始定價，徹底阻斷前端修改 Payload 竄改價格的攻擊路徑。

  3. 結帳與訂單處理
  - 結帳流程：強制會員登入驗證。建立完整的結帳表單 (`Checkout.jsx`) 與成功跳轉頁面 (`OrderSuccess.jsx`)。
  - 原子性交易防護 (Atomic Transactions)：運用 `DB::transaction` 確保訂單建立、明細快照寫入與購物車清理三者處於同一個原子操作中，消滅資料不一致風險。
  - 單價快照 (Price Snapshot)：在結帳時，將商品當下售價與規格固定存入 `order_items` 表，確保歷史訂單紀錄不受未來商品價格波動影響。
  - 會員訂單查詢：導入專屬的 `/member/orders` 獨立路由。精準設定 Model 關聯 (`Order` -> `OrderItem` -> `Product`) 以解決深層資料取得 (Eager Loading)，並在前端完美呈現歷史購買紀錄。

四、 開發工具、協作模式與環境調校
  - AI 合約式協作 (Contract-based Development)：引入「事前資源相容性驗證」策略。透過「架構生成端」與「全文檢索端」雙重 AI 工具互相核對資源庫，大幅降低修改遺漏與資料表欄位衝突的風險。
  - 建立專屬通道：開發 `clean_for_lm.php` 腳本，將 Laravel 與 React 專案檔案標準化輸出為純文字。利用 HTML 實體編碼迴避 Markdown 解析器對特殊符號 (`&#96;`, `&#36;`) 的誤判。
  - 編譯器最佳化：移除已棄用的 `jsconfig.json` `baseUrl` 配置，改採現代化的 `@/*` alias 映射，消除 TypeScript/Vite 路徑解析警告。
